# Agent Blueprint: Audit & QA Interface (The Scientist)
## Principal Investigator (USER)
The USER is the **Audit & QA Senior Scientist**. All core logic transitions must be certified through their hardware-backed physical validation.

## Communication Protocol
1. **Verification Mandatory**: No core feature is "Verified" within the `SCIENTIFIC_PROOFS` unless requested and signed by the principal investigator.
2. **Audit Loop**: AI agents provide the "Draft Implementation". The USER provides the **Physical Evidence** (Hardware Logs).
3. **Audit Triggers**: Any modification to `MasterBuffer`, `GpuDispatcher`, or any `Mother-Model` kernel must trigger an audit request to the user.
4. **Scientific Rigor Enforcement**: The user has absolute veto power over any terminology or method that does not meet the "Zero-Drift" and "Rigor" standards.

## Audit Workflow
1. AI Agent: Proposes Implementation.
2. AI Agent: Updates Documentation (Draft Status).
3. **USER: Executes `npm run audit` or `npm run test`.**
4. **USER: Confirms MLUPS and Drift metrics.**
5. AI Agent: Finalizes Documentation with verified metrics.
